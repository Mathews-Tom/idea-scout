# Lean Canvas Guide

Reference for constructing and evaluating Lean Canvas frameworks.

## Canvas Blocks

| Block | Question | Output |
|-------|----------|--------|
| Problem | What are the top 3 problems? | Ranked problem list with severity |
| Customer Segments | Who experiences these problems? | Specific personas with demographics |
| Unique Value Proposition | Why is this solution different? | Single clear sentence |
| Solution | What solves the top 3 problems? | Feature mapped to each problem |
| Channels | How do you reach customers? | Acquisition channels ranked by cost |
| Revenue Streams | How does it make money? | Revenue model with pricing |
| Cost Structure | What are the major costs? | Fixed vs. variable, burn rate |
| Key Metrics | What numbers matter? | 3-5 KPIs with targets |
| Unfair Advantage | What cannot be easily copied? | Defensibility assessment |

## Problem Validation Criteria

| Signal | Strong | Weak |
|--------|--------|------|
| Frequency | Daily or weekly occurrence | Rare, situational |
| Severity | Blocks workflow, causes financial loss | Minor inconvenience |
| Existing solutions | Workarounds are painful or expensive | Good-enough solutions exist |
| Willingness to pay | Customers actively seeking solutions | No purchase intent |
| Addressable | Solvable with technology | Systemic, cultural, or regulatory |

## Revenue Model Types

| Model | Best For | Key Metric |
|-------|----------|------------|
| SaaS subscription | Recurring value, B2B | MRR, churn rate |
| Marketplace | Network effects, two-sided | GMV, take rate |
| Transactional | Per-use value | Transaction volume, AOV |
| Freemium | High volume, low marginal cost | Conversion rate, ARPU |
| Usage-based | Variable consumption | Usage growth, revenue per unit |
| Licensing | Enterprise, on-premise | Deal size, renewal rate |
| Advertising | High traffic, consumer | DAU, CPM, CTR |

## Unfair Advantage Assessment

| Advantage Type | Durability | Example |
|----------------|-----------|---------|
| Network effects | High | More users = more value |
| Proprietary data | High | Unique dataset competitors lack |
| Domain expertise | Medium | Deep industry knowledge |
| Existing audience | Medium | Built-in distribution |
| First mover | Low-Medium | Only if combined with execution speed |
| Technology | Low | Can be replicated |
| Brand | Grows over time | Trust, recognition |

## Canvas Quality Checks

| Check | Requirement |
|-------|-------------|
| Problem-solution fit | Each solution maps to a specific problem |
| Customer specificity | Target customer is a persona, not "everyone" |
| UVP clarity | Understandable in one sentence, no jargon |
| Revenue realism | Pricing validated against willingness to pay |
| Channel feasibility | At least one channel the team can execute on now |
| Cost completeness | Includes acquisition cost, not just build cost |
| Metrics actionable | Each metric has a target and measurement method |
| Unfair advantage honest | "None yet" is acceptable; "passion" is not |

## Riskiest Assumption Identification

Rank assumptions by:
1. **Impact if wrong:** Would this invalidate the entire business?
2. **Uncertainty level:** How confident are we this is true?
3. **Testability:** Can we validate this cheaply and quickly?

Highest risk = high impact + high uncertainty + testable. Test these first.
